## 🗄️ 데이터 API 정리

### 사용 중인 API (3개)

#### 1. 의안정보 통합 API
**제공 데이터**:
- 의안 기본 정보 (제목, 날짜, 제안자)
- 진행 단계
- 소관위원회
- 상세 링크

**활용 화면**: 의안 목록, 의안 상세

---

#### 2. 국회의원 본회의 표결정보 API
**제공 데이터**:
- 의안별 표결 결과
- 의원별 표결 정보 (찬성/반대/기권/불참)
- 표결 일시

**활용 화면**: 의안 상세 (표결 현황), 의원 상세 (관심 분야 분석)

---

#### 3. 국회의원 정보 통합 API
**제공 데이터**:
- 의원 기본 정보 (이름, 정당, 선거구)
- 소속 위원회
- 당선 대수
- 의원 사진
- 연락처 정보 (일부만 제공)

**활용 화면**: 의원 상세, 유사 성향 의원

## 🗃️ 데이터베이스 테이블 정리

### 핵심 테이블 (4개)

#### 1. bills (의안)
**데이터 소스**: 의안정보 통합 API

**주요 필드**:
- `bill_id`: 의안ID
- `title`: 의안명
- `proposal_date`: 제안일
- `proposer_name`: 제안자명
- `proc_stage_cd`: 진행 단계
- `summary`: AI 요약
- `categories`: 카테고리 (JSON)
- `link_url`: 상세 링크

**용도**: 의안 목록, 의안 상세, 유사 의안 추천

---

#### 2. assembly_members (국회의원)
**데이터 소스**: 국회의원 정보 통합 API

**주요 필드**:
- `member_id`: 의원코드
- `name`: 의원명
- `party`: 정당명
- `district`: 선거구
- `committee`: 소속위원회
- `photo_url`: 사진 URL
- `phone`: 전화번호 (일부만)
- `email`: 이메일 (일부만)
- `homepage_url`: 홈페이지 (일부만)
- `office_room`: 사무실 호실 (일부만)
- `aide_name`: 보좌관 (일부만)
- `secretary_name`: 비서관 (일부만)

**용도**: 의원 상세, 유사 성향 의원

---

#### 3. votes (표결)
**데이터 소스**: 국회의원 본회의 표결정보 API

**주요 필드**:
- `vote_id`: 표결ID
- `bill_id`: 의안ID (FK)
- `member_id`: 의원ID (FK)
- `vote_result`: 표결결과
- `vote_date`: 표결 일시

**용도**: 의안별 표결 현황, 의원별 관심 분야 분석

---

#### 4. user_votes (사용자 투표)
**데이터 소스**: 자체 구현

**주요 필드**:
- `user_vote_id`: 투표ID
- `bill_id`: 의안ID (FK)
- `user_id`: 사용자 식별자
- `vote_result`: 투표 결과 (찬성/반대)

**용도**: 사용자 투표 기능, 투표 통계

---

### 추가 테이블 (3개)

#### 5. proc_stage_mapping (진행 단계 매핑)
**용도**: 진행 단계 코드를 순서로 변환

**주요 필드**:
- `stage_code`: 진행 단계 코드
- `stage_name`: 진행 단계 이름
- `stage_order`: 진행 단계 순서

---

#### 6. member_interests (의원 관심 분야) - 추후

**주요 필드**:
- `member_id`: 의원ID (FK)
- `category`: 카테고리
- `interest_score`: 관심도 점수

---

#### 7. user_political_profile (사용자 정치성향) - 추후
**용도**: 사용자의 정치성향 테스트 결과 저장

**주요 필드**:

- `p_score`, `m_score`, `u_score` 등: 정치성향 점수

## 📋 구현 우선순위

### Phase 1: MVP (즉시 시작 가능)

**의안 관련**:
1. 의안 목록 조회
2. 의안 상세 정보 표시
3. AI 요약 표시
4. 심사 진행 단계 표시
5. 사용자 투표 기능
6. 유사 의안 추천 (카테고리 기반)

**의원 관련**:
1. 의원 상세 정보 표시
2. 대표 발의 의안 목록

**예상 기간**: 2-3주

---

### Phase 2: 추가 기능 (선택사항)

1. 전체 발의자 수 크롤링
2. 의원 관심 분야 분석 (표결 이력 기반)
3. 의원별 표결 통계

**예상 기간**: 1주

---

### Phase 3: 고급 기능 (추후)

1. 사용자 정치성향 테스트
2. 사용자-의원 일치율 계산
3. AI 임베딩 기반 유사 의안 추천

**예상 기간**: 2-3주

## ⚠️ 주의사항

### 데이터 부족 가능성

**의원 연락처 정보**:
- 전화번호, 이메일, 홈페이지는 일부 의원만 제공
- **대응**: 데이터가 없는 경우 "-" 또는 "정보 없음" 표시

**보좌관/비서관 정보**:
- 일부 의원만 제공
- **대응**: 데이터가 없는 경우 "-" 표시

**전체 발의자 수**:
- API에서 대표 발의자만 제공
- **대응**: "이수진 의원 등" 형태로 표시하거나 크롤링
