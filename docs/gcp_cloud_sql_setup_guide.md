# Cloud SQL 설정 가이드 (무료 크레딧 최적화)

## ✅ 추천 설정

### 1. Cloud SQL 버전
- **Enterprise** 선택 (Enterprise Plus는 비쌈)
  - 99.95% 가용성으로 충분
  - 무료 크레딧 절약

### 2. 버전 사전 설정
- **"개발/테스트"** 또는 **"프로덕션"** 선택
  - 개발/테스트: 더 저렴
  - 프로덕션: 안정적 (무료 크레딧으로 충분)

### 3. 인스턴스 정보
- **인스턴스 ID**: `mypoly-postgres` (소문자, 하이픈 사용 가능)
- **비밀번호**: 강력한 비밀번호 설정 (기록 필수!)

### 4. 머신 유형 (중요!)
현재 화면에서 **"머신 유형"** 섹션을 찾아서:
- **db-f1-micro** 또는 **db-g1-small** 선택
  - db-f1-micro: 가장 저렴 (무료 크레딧으로 충분)
  - db-g1-small: 약간 더 성능 (여유 있으면)

### 5. 리전
- **asia-northeast3** (서울) 선택
  - 한국에서 가장 가까움
  - 현재 화면: us-central1 → **asia-northeast3로 변경**

### 6. 저장용량
- **10GB** 또는 **20GB**로 시작
  - 나중에 확장 가능
  - 현재 250GB는 너무 큼

### 7. 가용성
- **단일 영역** 선택 (고가용성 불필요)
  - 비용 절약
  - 개발/테스트 단계에서는 충분

### 8. 백업
- **자동 백업** 활성화 (선택사항)
  - 무료 크레딧 여유 있으면 활성화
  - 없으면 나중에 활성화 가능

---

## ⚠️ 현재 화면에서 변경해야 할 것

1. **리전 변경**: `us-central1` → `asia-northeast3` (서울)
2. **머신 유형 변경**: `db-perf-optimized-N-8` → `db-f1-micro` 또는 `db-g1-small`
3. **저장용량 변경**: `250GB` → `10GB` 또는 `20GB`
4. **가용성 변경**: "여러 영역" → "단일 영역"
5. **Enterprise Plus** → **Enterprise** (선택사항, 비용 절약)
