# Figma 정보 공유 가이드

## 문제 상황
Figma에 화면이 너무 많아서 모두 공유하기 어려운 상황

## 추천 방법

### 방법 1: 핵심 화면만 선별 공유 (추천)
1. **주요 기능별 대표 화면 선택**
   - 메인 화면
   - 의안 목록 화면
   - 의안 상세 화면
   - 투표 화면
   - 국회의원 표결 현황 화면

2. **각 화면별로 필요한 데이터 정리**
   - 화면에 표시되는 정보
   - 필요한 API 데이터
   - 구현 가능 여부

### 방법 2: 기능별 그룹화하여 공유
1. Figma에서 화면을 기능별로 그룹화
2. 각 그룹별로 프로토타입 링크 생성
3. 그룹별로 필요한 데이터 요구사항 정리

### 방법 3: 요구사항 문서 작성 후 공유
1. Figma 화면을 보면서 각 화면의 **데이터 요구사항**만 정리
2. 예시:
   ```
   화면: 의안 상세 페이지
   - 의안 제목
   - 의안 내용 (원문 또는 요약)
   - 제안자 정보
   - 현재 진행 단계
   - 관련 위원회 정보
   - 표결 현황 (찬성/반대/기권)
   ```

## 제안하는 워크플로우

1. **1단계: 핵심 화면 선별**
   - 기획자/디자이너와 함께 핵심 화면 5-10개 선별
   - 각 화면의 스크린샷 또는 Figma 링크 수집

2. **2단계: 데이터 요구사항 정리**
   - 각 화면에 표시되는 데이터 항목 리스트업
   - 이 문서에 정리

3. **3단계: API 검토 및 매핑**
   - 정부 API에서 제공하는 데이터 확인
   - 화면 요구사항과 API 데이터 매핑
   - 구현 가능 여부 판단

4. **4단계: 공유 문서 작성**
   - 검토 결과를 기획자/디자이너가 이해하기 쉽게 정리
   - 구현 불가능한 기능이 있다면 대안 제시

## 다음 단계

기획자/디자이너분께 다음을 요청해주세요:
1. 핵심 화면 5-10개 선별
2. 각 화면의 스크린샷 또는 Figma 링크 공유
3. 또는 각 화면에 표시되어야 할 데이터 항목 리스트
